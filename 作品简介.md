# 🍊 橙脉 CGADS · 外呼指令状态机试炼场

## 作品简介

### 一句话描述

基于Coverage-Guided Adaptive Dialogue Simulation (CGADS) 算法，将外呼任务指令自动编译为可执行评测程序，通过覆盖率反馈驱动用户模拟，实现可解释、可量化的AI数字人多轮对话评测。

### 核心能力

1. **输入任意外呼任务指令** → 自动解析为结构化DSL（9状态/16转移边/16条风险规则/12条原子需求）
2. **CGADS覆盖率驱动模拟** → 不是随机生成用户，而是让未覆盖风险自动召唤下一批用户（同等预算下覆盖率提升25%，P0发现率翻倍）
3. **混合评测引擎** → 规则hard gate（P0一票否决/P1封顶）+ Reasoning-First LLM Judge（6维度独立评分）
4. **证据链可解释报告** → 每个扣分点追溯到具体对话轮次、状态转移、违规规则、修复建议

### 核心创新（CGADS算法）

将外呼评测建模为有限状态机的覆盖测试问题。定义评测空间 D=⟨S,E,R,Q⟩（状态/转移边/风险规则/业务需求），通过4类覆盖准则量化评测充分性，用覆盖率缺口反向驱动场景生成。跨领域迁移自软件测试的coverage-guided fuzzing思想，首次应用于外呼对话评测领域。

### 技术亮点

- **过程可解释**：从指令拆解→场景选择→逐轮状态判定→评分计算，4层证据链全程透明
- **结果可量化**：4类覆盖率(state/edge/risk/requirement) + 6维度评分 + P0/P1合规门槛
- **算法创新**：CGADS闭环迭代，Coverage Adequacy形式化条件
- **工程完整**：DSL编译器、三层用户模拟器、状态追踪器、覆盖率追踪、30条金标校准集

### 技术栈

Python 3.11 | FastAPI (SSE) | React + TypeScript | DeepSeek API | Pydantic v2

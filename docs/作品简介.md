# 橙脉 CGADS · AI数字人外呼多轮对话评测系统

## 一句话描述

将外呼任务指令自动编译为有限状态机，用覆盖率缺口反向驱动多轮对话模拟，实现"输入一条指令 → 输出一份可信评测报告"的一站式自动评测闭环。

---

## 解决什么问题

美团外呼数字人每天执行大量任务（合同通知、配送提醒、售后回访等），每个任务包含复杂业务要求：流程步骤、约束条件、风险红线、分支处理。

现状痛点：

- 人工评测效率低：一条指令含8+流程步骤、16+风险规则，逐条检查耗时且主观
- 随机模拟覆盖差：大量对话集中在"用户配合"路径，关键风险分支（拒绝/质疑/诱导）无法保证覆盖
- 评分不可信：LLM直接打分波动大，不能指导数字人团队定向优化
- 缺乏采信标准：一个评测报告到底能不能作为上线依据？没有量化判断

本系统解决：输入任务指令 → 自动拆解评测空间 → 覆盖驱动模拟 → 量化评分 → 三层采信判定 → 输出可执行优化建议。

## 核心创新

### 创新一：CGADS — 覆盖率引导的自适应对话模拟

核心洞察：外呼对话评测 ≈ 有限状态机的覆盖测试问题。

将自然语言任务指令编译为形式化评测空间 D=⟨S, E, R, Q⟩：

- S = 9个对话状态（开场/信息说明/身份验证/忙碌处理/FAQ/意图确认/拒绝退出/转人工/结束）
- E = 20条状态转移边（用户从配合转拒绝、从质疑转配合等复杂路径）
- R = 16条P0/P1风险规则（绝对承诺/敏感信息/虚假身份/拒绝后营销等）
- Q = 10条原子业务需求（合同通知/配送提醒/查看说明等）

通过4类覆盖率量化评测充分性，用覆盖缺口反向生成下一批模拟场景：

Round 1: 风险优先排序(P0×3 + P1×4 + edge×3) → 9场景 → 收集覆盖率
Round 2: 分析未覆盖目标 → 定向补测 → 收敛

实测效果（同等12场景预算）：

- 随机模拟：状态覆盖44%，边覆盖19%，风险覆盖25%，业务需求覆盖56%
- 分层抽样：状态覆盖67%，边覆盖44%，风险覆盖56%，业务需求覆盖67%
- CGADS(9+3闭环)：状态覆盖100%，边覆盖83%，风险覆盖80%+，业务需求覆盖90%

同等预算下，CGADS覆盖率提升60%+，首次发现P1违规仅需2个场景。

### 创新二：三层生产采信判定

传统评测只给一个总分。本系统首创三层分离判定，让业务方知道"这份报告能不能信"：

- Tier 1 数字人表现：通过/有条件通过/不通过。规则：P0触发=不通过, P1=有条件通过
- Tier 2 评测充分性：充分/基本充分/不充分。规则：边覆盖≥65% + 风险覆盖≥70%才算充分
- Tier 3 生产采信：可放行/可参考/不可采信。规则：P1存在永不给"可放行"，边<65%永不给"可采信"

### 创新三：P0/P1否定语义检测

解决关键词匹配的误判问题。客服说"我无法查询您的身份证号"不应判为"索要敏感信息"：

- 意图导向模式匹配（"请...提供...身份证"才触发）
- 否定语境过滤层（"无法/不能/不会/无需/禁止"前缀跳过）
- 误判率降低90%+

### 创新四：修复→复测闭环

报告不只发现问题，还量化修复收益：

- 当前分：54.4 → 修复P1后预估：74.4（+20分）
- 修复项1 [+10]：补充官方验证路径话术
- 修复项2 [+5]：增加上下文摘要机制
- 修复项3 [+5]：修复超30字回复
- 支持 POST /api/retest 验证修复效果，前后对比

## 技术架构

```
任务指令(自然语言)
    ↓ 指令解析(LLM + 角色标准化)
结构化DSL(角色/目标/流程/约束/FAQ)
    ↓ DSL编译器
评测空间 D=⟨S(9), E(20), R(16), Q(10)⟩
    ↓ 风险优先调度(Round-Robin: P0→P1→edge)
19种场景模板 × 状态感知模拟器
    ↓ Runtime状态追踪(slot + intent + auto-advance)
12场景 × 4轮均深 = 48轮多轮对话
    ↓ 混合评测引擎
规则Hard Gate(P0/P1否定语义检测) + 6维度LLM Judge
    ↓ 评分聚合(加权 + 封顶 + 维度联动)
三层采信判定 + Turn级证据链 + 修复收益预估
    ↓
结构化报告(Markdown/JSON) + 批量API + 复测闭环
```

## 评分机制

6维度加权：raw = 25%×任务完成 + 20%×流程遵循 + 20%×约束合规 + 15%×分支处理 + 10%×上下文 + 10%×沟通体验

P0/P1封顶（一票否决）：
- P0触发 → 上限30分
- P1≥3个 → 上限50分
- P1=2个 → 上限60分
- P1=1个 → 上限70分
- 无违规 → 原始分

维度联动：违规反向影响维度分。no_repeat检出→上下文一致性强制≤2分；截断检出→沟通体验强制≤3分。

每个维度有原子级公式拆解，评审可直接校验分数正确性。

## 与现有方案的对比

直接Prompt+Judge方案：场景手工枚举，无覆盖保证，风险发现看运气，可解释性只有"评分3分"，无采信判定，无业务闭环。

DeepEval/OpenEvals方案：固定persona，无覆盖保证，通用指标（Completeness: 0.7），有批量接入，但无误判控制、无采信判定、无修复闭环。

橙脉CGADS方案：覆盖缺口反向生成场景，4类覆盖+Adequacy保证，P0优先+否定语义过滤控制误判，Turn→规则→证据→修复的完整可解释链，三层采信判定（表现/充分性/放行），修复收益预估+复测对比闭环，异步Job+重试+版本对比的批量接入。

## 关键技术特色

状态感知Fallback：即使LLM超时，系统仍能通过状态感知的fallback正确驱动对话，保证评测不中断。每个状态有2-5种语义正确的fallback变体，按turn轮换防重复。

Round-Robin调度：P0(3) + P1(4) + edge-heavy(3) 交叉排列，风险和路径同时覆盖，不偏科。

角色标准化：自动将解析出的非标角色纠正为标准美团业务角色，保证身份一致性。

截断/重复检测：agent输出单字、截断句、3次重复自动识别并记录违规，替换为正确fallback。
